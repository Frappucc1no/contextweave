# RecallLoom v0.3.5

RecallLoom v0.3.5 improves day-to-day project continuity with faster resume, structured progress capture, and safer managed updates.

## What Is New

- Faster resume for existing projects through compact current-state snapshots before deeper context loading is needed.
- Structured milestone appends that keep daily-log entry ordering clear within each daily log file.
- Previewable managed writes for project background, current state, and daily progress updates.
- Clearer recovery guidance when a write cannot be completed, including the next safe action to take.
- Safer attached-text handling before continuity content is shared with connected tools.

## Compatibility

- Package version: `0.3.5`
- Sidecar protocol: `1.0`
- Minimum Python version: `3.10`
- Supported workspace languages: English and Simplified Chinese
- Existing RecallLoom projects using protocol `1.0` do not need a sidecar migration.

## Upgrade

Use your normal skill update flow. With a Skills CLI, run:

```bash
npx skills update
```

For directory-based installs, replace the installed `recallloom/` skill directory with the package copy from this release.

If you use native command wrappers, rerun `install_native_commands.py` after updating the package.

## Validation Summary

This release was validated with the RecallLoom unit test suite on the supported local Python runtimes, repository whitespace checks, and public release metadata checks.
